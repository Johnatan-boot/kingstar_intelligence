✅ 2. Contrato de Interface (JSON com backend)

Aqui vamos padronizar como seu frontend conversa com o backend.

📦 PADRÃO GLOBAL DE RESPOSTA
{
  "success": true,
  "data": {},
  "message": "",
  "meta": {}
}
📌 EXEMPLO REAL — Módulo Purchases
🔽 GET /purchases
{
  "success": true,
  "data": [
    {
      "id": "p1",
      "productName": "Mouse Gamer",
      "quantity": 5,
      "price": 120.50,
      "supplier": "Fornecedor A",
      "status": "pending",
      "createdAt": "2026-04-08T10:00:00Z"
    }
  ],
  "meta": {
    "total": 50,
    "page": 1,
    "limit": 10
  }
}
🔼 POST /purchases
{
  "productName": "Mouse Gamer",
  "quantity": 5,
  "price": 120.50,
  "supplier": "Fornecedor A"
}
❌ ERRO PADRÃO
{
  "success": false,
  "message": "Erro de validação",
  "errors": [
    {
      "field": "quantity",
      "message": "Quantidade deve ser maior que zero"
    }
  ]
}
🎯 Decisões importantes:
data sempre contém dados
meta para paginação
errors para formulário
success evita ifs confusos no frontend
✅ 3. Escolha de Estado (DECISÃO CERTA PRA VOCÊ)

Agora o ponto mais importante.

Seu projeto:

múltiplas páginas
dados vindos de API
interação de UI

👉 Então você NÃO usa só um tipo de estado — usa combinação.

🧠 Arquitetura de Estado IDEAL
🟢 1. Estado LOCAL (useState)

✔ Use para:

abrir modal
inputs
controle visual
const [isModalOpen, setIsModalOpen] = useState(false);
🔵 2. Estado de API (PRINCIPAL)

👉 Esse é o mais importante do seu sistema

✔ Use:

React Query (TanStack Query)
const { data, isLoading } = useQuery({
  queryKey: ['purchases'],
  queryFn: getPurchases
});

✔ Resolve:

cache
loading
refetch automático
sincronização
🟡 3. Estado GLOBAL (leve)

👉 Use Zustand, não Redux

✔ Para:

usuário logado
permissões
tema
pequenas infos globais
const useStore = create((set) => ({
  user: null,
  setUser: (user) => set({ user })
}));
🟣 4. Estado via URL

✔ Para:

paginação
filtros
busca
/purchases?page=1&status=pending
🎯 RESUMO FINAL (DECISÃO PROFISSIONAL)
Tipo de estado	Ferramenta	Uso
Local	useState	UI
API	React Query	Dados principais
Global	Zustand	Sessão/usuário
URL	React Router	Filtros
